Généralités
L'outil IA du groupe BETC, mis à disposition de ses équipes. Un environnement sécurisé pour accéder aux meilleurs modèles du marché, GPT, Claude, Gemini, sans exposer vos données ni celles de vos clients. Contrairement aux outils publics, vos conversations ne servent pas à réentraîner les modèles. Ce que vous tapez ici reste ici.
Bonne pratique. Même dans un environnement sécurisé, pensez toujours à anonymiser les données clients avant de les utiliser dans vos prompts.
Tokens et budget
En pratique, un échange standard coûte entre $0.01 et $0.05. Une conversation longue avec un document dense peut monter à $0.30. Le token est l'unité de mesure derrière ces coûts : chaque mot envoyé ou reçu consomme des tokens, et le prix varie selon le modèle utilisé.
Astuce. Un modèle plus performant consomme plus de tokens par échange. Si votre consommation semble élevée, vérifiez d'abord le modèle choisi et la configuration de vos assistants.
Un crédit mensuel en dollars alloué à votre compte pour couvrir les tokens consommés. Il s'affiche en temps réel en bas de l'interface. Budget standard : $10 par mois, réinitialisé chaque mois.
Les modèles basiques sont souvent aussi performants que les dernières versions, y compris pour des tâches semi-complexes, et consomment nettement moins de budget.
⚠ Attention. Les fichiers de référence (RAG) de vos assistants sont retraités à chaque échange. Un assistant chargé de fichiers lourds consommera significativement plus de budget qu'un assistant léger. Pour limiter la consommation, précisez dans votre message quel fichier consulter (ex : "en te basant sur le fichier X, ...") plutôt que de laisser l'assistant scanner tout le corpus.

Les modèles
Trois familles disponibles, trois profils d'usage :

GPT (OpenAI) : la majorité des tâches quotidiennes. Polyvalent, rapide, bien calibré pour la rédaction et l'analyse standard.

Claude (Anthropic) : privilégiez-le pour les tâches complexes, documents longs, raisonnement nuancé, écriture à fort enjeu éditorial.

Gemini (Google) : tâches rapides et répétitives, analyse de visuels, recherche web intégrée.

Règle simple : commencez par GPT. Montez en puissance si le résultat ne vous satisfait pas.
Le curseur va de 0 à 1.5 et contrôle le degré d'imprévisibilité des réponses :
  • 0 à 0.3 : réponses précises et factuelles, idéal pour l'analyse, les résumés, les calculs.
  • 0.4 à 0.7 : équilibre entre fiabilité et originalité, rédaction, conversation, recommandations.
  • 0.8 à 1.5 : idées variées et inattendues, brainstorming, concepts créatifs, exploration.
Le modèle prend le temps de structurer son raisonnement avant de répondre. Les réponses sont plus précises sur les sujets complexes. En contrepartie : plus lent, et plus gourmand en tokens. À réserver aux tâches qui le méritent.
Les assistants
  1. Cliquez sur "+ Créer un nouvel assistant"
  2. Renseignez le nom, la catégorie (Créa / Stratégique / Production / Média / Général), le fournisseur et le modèle
  3. Rédigez vos instructions dans le champ dédié, en markdown, en suivant une structure claire
  4. Cliquez sur "Créer"
  5. Une fois sauvegardé, vous pouvez uploader vos fichiers de référence
Ces étapes sont détaillées et illustrées pas à pas dans le How To. Pour maîtriser la rédaction de prompts et d'instructions, suivez la formation Prof IA → Lire l'article : Créer un assistant de A à Z
Le corpus documentaire de votre assistant, charte de marque, grille tarifaire, méthodologie interne. L'assistant s'y réfère pour répondre plutôt que de s'appuyer uniquement sur ses connaissances générales. Plus le corpus est précis, plus les réponses le sont.
L'assistant doit être sauvegardé avant de pouvoir uploader des fichiers. Limite : 10 fichiers maximum par assistant.
Attention à la consommation. Chaque fichier de référence est retraité à chaque échange, ce qui peut faire monter la facture rapidement, surtout avec des fichiers lourds (Excel, PDF volumineux). Ne chargez que les fichiers strictement nécessaires, privilégiez des fichiers légers et bien structurés, et orientez l'assistant vers un ou deux fichiers spécifiques dans votre prompt plutôt que de le laisser scanner tout le corpus.
Deux niveaux de mémoire par assistant :
  • Mémoire partagée : visible par toutes les personnes qui ont accès à l'assistant. Utile pour stocker des règles ou du contexte commun.
  • Mes mémoires : personnelles, propres à votre usage de cet assistant. Non visibles par les autres.
Via le bouton de partage en bas de la configuration. Une fois partagé, l'assistant apparaît dans la Marketplace pour vos collègues. Vous pouvez aussi partager une conversation spécifique : en cours d'échange, ouvrez les trois points (···) de la conversation pour choisir de la partager. Côté assistant, la vue « Conversations partagées » rassemble tous ces échanges.
Chaque modification crée une version horodatée. Accessible via le bouton "Version" dans la configuration, vous pouvez restaurer n'importe quelle version précédente à tout moment.
Les skills
Un skill est une instruction réutilisable que vous pouvez attacher à un assistant pour lui donner une compétence spécifique (un format de restitution, une méthodologie, un formalisme client). Vous pouvez les créer depuis la section Skills, les partager par export/import, et les activer sur un assistant via ses Outils avancés. Il est aussi possible d'invoquer un skill directement dans une conversation en tapant $ dans la barre de saisie.
Fonctionnalité avancée, en cours de stabilisation. Pour la majorité des usages, les instructions du system prompt suffisent.
Bugs et support
Utilisez le bouton Signaler un problème dans l'interface, ou remplissez directement le formulaire dédié. Renseignez :
  • Description du problème
  • Navigateur utilisé
  • Capture d'écran
  • Message d'erreur exact
Plus c'est précis, plus c'est rapide à corriger. Ouvrir le formulaire de signalement

How To › Créer un assistant de A à Z

Créer un assistant de A à Z

Un assistant, c'est un collaborateur sur mesure : vous définissez sa mission, son savoir et sa façon de répondre, une fois pour toutes.

Vue d'ensemble

La création d'un assistant se fait dans un panneau unique, découpé en trois temps : l'identité (qui il est), le moteur (comment il réfléchit) et le savoir (ce sur quoi il s'appuie).

Étape par étape

1Lancer la création

Dans le panneau de gauche, cliquez sur + Créer un nouvel assistant. Un formulaire vierge s'ouvre. Vous retrouverez tous vos assistants existants via Mes assistants, et ceux partagés par l'agence via la Marketplace.

2Donner une identité

Renseignez le nom et une description. Choisissez une catégorie (Créa, Stratégique, Production, Média ou Général).

Astuce. Un nom explicite vaut mieux qu'un nom malin. « Générateur de piliers éditoriaux » se retrouve, « ÉditoBot 3000 » se perd.
3Choisir le moteur

Sélectionnez le fournisseur puis le modèle.

  • GPT pour la majorité des tâches : rapide, polyvalent.
  • Claude pour le complexe : documents longs, nuance éditoriale.
  • Gemini pour le rapide et le visuel.
4Régler le niveau de créativité

Trois réglages complémentaires pour affiner le comportement du moteur.

5Rédiger les instructions (system prompt)

C'est le coeur de l'assistant. Une structure simple et éprouvée :

# RÔLE Qui est l'assistant, à qui il s'adresse. # OBJECTIF Sa mission en une phrase active. # CONTEXTE ET RÉFÉRENCES Priorise les fichiers fournis. # TÂCHES Ce qu'il fait, étape par étape. # LIMITES Ce qu'il ne fait pas, où rediriger. # FORMAT Langue, longueur, structure, ton.
Restez sous ~400 mots. Au-delà, le modèle dilue les premières consignes.
6Nourrir l'assistant : fichiers de référence (RAG)

Les fichiers de référence forment le corpus documentaire de l'assistant. Le system prompt décrit le comportement, le RAG porte la connaissance. Limite : 10 fichiers maximum par assistant.

Attention. Les fichiers de référence sont retraités à chaque message, ce qui impacte directement votre consommation de tokens. 10 fichiers Excel chargés = un coût multiplié à chaque échange. Chargez uniquement ce dont l'assistant a besoin et précisez dans vos instructions quels fichiers consulter en priorité.
7Gérer la mémoire

Deux niveaux de mémoire :

  • Mémoire partagée : visible par tous ceux qui ont accès à l'assistant.
  • Mes mémoires : personnelles, propres à votre usage.
8Versionner et restaurer

Chaque modification crée une version horodatée. Ouvrez l'historique via le bouton Version et restaurez n'importe quel état précédent.

9Partager avec l'équipe

Partagez via le bouton dédié : l'assistant rejoint la Marketplace et devient utilisable par vos collègues.

Bonne hygiène de partage. Limitez les droits de gestion à une ou deux personnes. Laissez les utilisateurs finaux en simple lecture. Gardez le corpus RAG à jour.
How To › Maîtriser les skills

Maîtriser les skills

Si un assistant est un collaborateur, un skill est une compétence qu'il peut mobiliser. Vous l'écrivez une fois, il s'applique partout. C'est la façon la plus efficace de capitaliser votre savoir-faire métier.

Le principe

Un skill est une instruction persistante, réutilisable dans toutes vos conversations. Là où le system prompt d'un assistant est figé dans cet assistant, un skill est transversal : une fois créé, il devient un outil que n'importe quel assistant peut solliciter selon le besoin.

La fenêtre de création d'un skill

Créer un skill

1

Ouvrir le créateur

Depuis la section Skills de la navigation, cliquez sur + Créer un skill. Trois champs vous attendent : nom, description, instructions.

2

Remplir les trois champs

  • Nom : explicite, car c'est ce que le modèle repère pour décider d'activer le skill.
  • Description : quand et pourquoi l'utiliser, en une phrase.
  • Instructions : la méthode, le format de sortie, les exemples, en markdown.

Bien rédiger un skill

La règle d'or : une compétence par skill. Un skill n'est pas un assistant miniature, c'est une brique unitaire.

  • Visez le précis : « format de restitution d'audit SEO » plutôt que « aide marketing ».
  • Décrivez le savoir-faire, pas juste la tâche : c'est votre manière de faire qui a de la valeur.
  • Donnez un exemple de sortie attendue : il vaut dix lignes de description.
Par où commencer ? Listez les tâches que vous répétez chaque semaine avec les mêmes exigences de format. Ce sont vos premiers skills, ceux qui vous feront gagner le plus de temps.

Importer et partager

Un skill se partage par export de fichier. Formats acceptés à l'import :

  • un fichier .md avec nom et description en YAML ;
  • un fichier .zip ou .skill contenant un SKILL.md.

Votre collègue clique sur Importer un skill, dépose le fichier, et la compétence rejoint son espace. C'est ainsi qu'une bonne pratique individuelle devient un standard d'équipe.

Activer un skill sur un assistant

Dans les Outils avancés de l'assistant, activez l'interrupteur Skills, puis Add Skills pour choisir ceux à mobiliser.

Note. La fonctionnalité skills est en cours de stabilisation. En attendant qu'elle soit pleinement active, vous pouvez toujours intégrer la méthode directement dans les instructions de l'assistant.

Le prompt engineering

Savoir parler à un modèle IA, ça s'apprend. Plutôt que de survoler le sujet ici, on vous a préparé une vraie formation.

Formation recommandée
Prof IA

La formation complète pour maîtriser le prompt engineering : structure, ton, exemples, itération, garde-fous. Indispensable avant de créer vos premiers assistants.

Accéder à la formation →

En résumé

Un bon prompt repose sur trois principes : être structuré (rôle, objectif, tâches, limites, format), être direct (impératif, positif, concret) et poser des garde-fous (périmètre, confidentialité, gestion de l'ambiguïté). Visez moins de 400 mots pour un system prompt.

Le system prompt décrit le comportement, les fichiers de référence portent la connaissance. Ne mélangez pas les deux.
